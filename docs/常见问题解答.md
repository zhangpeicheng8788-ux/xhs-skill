## 环境相关

### Q1: 需要什么环境？

**A**: 
- Windows 操作系统
- Python 3.14（推荐安装在 `C:\Python314\`）
- 依赖库：`xhs`, `python-dotenv`

### Q2: 如何安装 Python？

**A**:
1. 访问 https://www.python.org/downloads/
2. 下载 Python 3.14 安装包
3. 安装时勾选"Add Python to PATH"
4. 安装到 `C:\Python314\`（推荐）

### Q3: 如何安装依赖？

**A**:
```bash
# 方法1：使用环境测试工具（推荐）
双击运行: test_environment.bat

# 方法2：手动安装
C:\Python314\python.exe -m pip install xhs python-dotenv
```

**A**:
编辑所有 `.bat` 文件，将 `C:\Python314\python.exe` 改为你的 Python 路径。

例如：
```batch
# 原来
C:\Python314\python.exe scripts\publish_gui.py

# 修改为
C:\Users\YourName\AppData\Local\Programs\Python\Python314\python.exe scripts\publish_gui.py
```

## Cookie 相关

### Q5: 什么是 Cookie？为什么需要它？

**A**:
Cookie 是登录凭证，用于验证你的小红书账号身份。没有 Cookie 无法发布笔记。

### Q6: 如何获取 Cookie？

**A**:
详细步骤参考 `docs/Cookie获取指南.md`

简要步骤：
1. 访问 https://creator.xiaohongshu.com
2. 登录账号
3. 按 F12 → Network → 任意请求 → Headers → Cookie
4. 复制完整 Cookie 字符串
5. 粘贴到 `.env` 文件

### Q7: Cookie 多久失效？

**A**:
通常几小时到几天，具体取决于小红书的设置。失效后重新获取即可。

### Q8: Cookie 失效有什么提示？

**A**:
发布时会提示：
- "未找到 Cookie"
- "无登录信息"
- "Cookie 无效"

### Q9: Cookie 格式不正确怎么办？

**A**:
正确的 Cookie 格式：
```
a1=xxx; webId=xxx; web_session=xxx; ...（很长的一串）
```

常见错误：
- ❌ 只有一个 token 值（太短）
- ❌ 包含换行符
- ❌ 没有分号分隔

解决方案：
1. 重新复制完整的 Cookie
2. 确保复制的是 Request Headers 中的 Cookie
3. 确保没有多余的空格或换行

## 发布相关

### Q10: 为什么要间隔10分钟？

**A**:
避免被小红书平台识别为异常行为，防止限流或封号。

### Q11: 可以缩短间隔时间吗？

**A**:
可以，但不建议低于5分钟。修改方法：

编辑 `start_publish.bat`：
```batch
--wait-minutes 10    # 改为你想要的分钟数
```

### Q12: 发布失败了怎么办？

**A**:
1. 查看日志中的错误信息
2. 检查 Cookie 是否失效
3. 检查网络连接
4. 检查图片文件是否存在
5. 重新启动程序

### Q13: 如何从中途继续发布？

**A**:
编辑 `start_publish.bat`：
```batch
--start-from 1    # 改为你要继续的笔记编号
```

例如，从第5篇开始：
```batch
--start-from 5
```

### Q14: 发布的笔记在哪里查看？

**A**:
- 方法1：运行 `view_progress.bat` 查看
- 方法2：访问 https://creator.xiaohongshu.com
- 方法3：查看 `publish_records.json` 文件

### Q15: 如何知道笔记是否发布成功？

**A**:
- 发布工具会显示"发布成功"
- 会显示笔记ID和链接
- 进度查看器会显示已发布笔记
- `publish_records.json` 会记录发布信息

## GUI 工具相关

### Q16: 双击后没有反应？

**A**:
1. 等待几秒（程序启动需要时间）
2. 检查是否有错误提示窗口
3. 运行 `test_environment.bat` 检查环境
4. 查看是否有 Python 进程在运行

### Q17: 窗口一闪就消失？

**A**:
这通常是环境问题：
1. Python 未安装或路径不正确
2. 缺少依赖库
3. 脚本语法错误

解决方案：
1. 运行 `test_environment.bat` 诊断
2. 检查 Python 安装
3. 安装依赖库

### Q18: 可以关闭窗口吗？

**A**:
- ❌ 发布工具窗口：不能关闭，会中断任务
- ✅ 进度查看器窗口：可以关闭，不影响发布
- ✅ 可以最小化窗口

### Q19: 如何暂停发布？

**A**:
点击发布工具窗口中的"暂停"按钮。暂停后可以点击"继续"恢复。

### Q20: 如何停止发布？

**A**:
点击发布工具窗口中的"停止"按钮。停止后需要重新开始。

## 错误处理

### Q21: 提示"Python not found"

**A**:
Python 未安装或路径不正确。

解决方案：
1. 检查 Python 是否安装在 `C:\Python314\`
2. 如果安装在其他位置，修改 `.bat` 文件中的路径
3. 重新安装 Python

### Q22: 提示"Missing dependency"

**A**:
缺少依赖库。

解决方案：
```bash
C:\Python314\python.exe -m pip install xhs python-dotenv
```

或运行 `test_environment.bat` 自动安装。

### Q23: 提示"Syntax error"

**A**:
Python 脚本语法错误。

解决方案：
1. 重新下载 skill 文件
2. 不要修改 Python 脚本
3. 确保使用 Python 3.14

### Q24: 提示"未找到 Cookie"

**A**:
`.env` 文件不存在或 Cookie 未配置。

解决方案：
1. 在 skill 主目录创建 `.env` 文件
2. 添加内容：`XHS_COOKIE=你的Cookie`
3. 确保 Cookie 格式正确

### Q25: 提示"Cookie 无效"

**A**:
Cookie 已失效。

解决方案：
1. 重新获取 Cookie
2. 更新 `.env` 文件
3. 重新运行发布工具

### Q26: 提示"网络错误"

**A**:
网络连接问题。

解决方案：
1. 检查网络连接
2. 检查防火墙设置
3. 尝试使用代理
4. 稍后重试

### Q27: 图片上传失败

**A**:
可能原因：
- 图片文件不存在
- 图片格式不支持
- 图片太大
- 网络问题

解决方案：
1. 检查图片文件是否存在
2. 确保图片格式为 PNG
3. 检查图片大小（建议<5MB）
4. 检查网络连接

### Q28: 发布后看不到笔记

**A**:
可能原因：
- 笔记在审核中
- 笔记被设为私密
- 账号被限流

解决方案：
1. 等待审核（通常几分钟到几小时）
2. 检查笔记是否设为私密
3. 访问创作者中心查看状态

## 其他问题

### Q29: 如何修改发布参数？

**A**:
编辑 `start_publish.bat` 文件：
```batch
--notes-dir "路径"      # 笔记目录
--start-from 1         # 起始笔记编号
--wait-minutes 10      # 发布间隔（分钟）
```

### Q30: 如何查看发布记录？

**A**:
- 方法1：运行 `view_progress.bat`
- 方法2：打开 `publish_records.json` 文件
- 方法3：访问创作者中心

### Q31: 发布记录文件在哪里？

**A**:
```
D:\20260127XHS\Auto-Redbook-Skills-main\publish_records.json
```

### Q32: 如何备份发布记录？

**A**:
复制 `publish_records.json` 文件到其他位置。

**A**:
不建议。每个账号需要单独的 Cookie，建议分别发布。

### Q34: 如何更新 Skill？

**A**:
1. 备份 `.env` 文件（Cookie 配置）
2. 备份 `publish_records.json`（发布记录）
3. 下载新版本 Skill
4. 恢复备份的文件

### Q35: 遇到其他问题怎么办？

**A**:
1. 查看日志文件（发布工具会显示详细日志）
2. 查看 `SKILL.md` 完整文档
3. 查看 `CHANGELOG_V6.6.md` 版本说明
4. 运行 `test_environment.bat` 诊断环境

## 💡 使用技巧

### 技巧1：同时打开两个工具

同时运行：
- `start_publish.bat` - 查看发布过程
- `view_progress.bat` - 查看发布结果

### 技巧2：测试模式

修改间隔为1分钟进行测试：
```batch
--wait-minutes 1
```

测试成功后再改回10分钟。

### 技巧3：断点续传

如果发布中断，记录最后发布的笔记编号，然后：
```batch
--start-from N    # N为下一篇笔记编号
```

### 技巧4：定期更新 Cookie

建议每周更新一次 Cookie，避免发布时失效。

### 技巧5：备份重要文件

定期备份：
- `.env` - Cookie 配置
- `publish_records.json` - 发布记录
